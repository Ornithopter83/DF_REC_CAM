# 결정 기록

이 문서는 프로젝트에서 결정된 사항만 짧게 기록한다. 대화 요약을 장문으로 남기지 않는다.

## 형식

```text
YYYY-MM-DD
- 결정 내용
- 이유 또는 영향
```

## 2026-06-26

- 긴 대화 이주용 요약을 반복 작성하지 않는다.
- 프로젝트 맥락은 대화가 아니라 저장소 문서에 둔다.
- 작업은 `tasks/*.md` 단위로 나누어 수행한다.
- 작업 완료 후 새로 생긴 결정사항만 `docs/DECISIONS.md`에 짧게 추가한다.
- 녹화/재생 전용 배포 모드는 실행 인수 `--reconly`로 켠다.
- 기본형에 가까운 지역 변수는 우항 타입이 명확하면 `var`보다 명시 타입을 우선 사용한다.

## 2026-07-15

- UI 외관 개선에는 Krypton Toolkit을 도입한다.
- 기능/성능 개선이 아니라 기본 Windows Forms 앱 느낌을 줄이는 차분한 밝은 테마 적용을 목표로 한다.
- 영상 표시, 녹화, 감지, 재생 흐름은 UI 테마 변경과 분리해 유지한다.
- 상태 중심 UI는 승인된 디자인 시안의 정보 계층과 색상을 최대한 따르되 실제 제공되는 상태 데이터만 표시한다.
- 녹화 일시 정지, 실시간 수신 비트레이트, 네트워크 상태, 이벤트 배지, 저장 경로 변경처럼 현재 동작이 없는 항목은 UI 시안만을 근거로 새 기능화하지 않는다.
- 저장소 동작은 기존 `AppPaths.RecVideos` 정책을 유지하고 화면에는 실제 경로를 읽기 전용으로 표시한다.
- 메인 오른쪽 작업 영역은 녹화와 파일 재생을 별도 탭으로 분리하며, 영상 불러오기는 재생 탭에서만 제공한다.
- 전체화면에서는 영상 외의 상단 헤더, 오른쪽 작업 영역, 하단 상태바와 레이아웃 여백을 모두 숨긴다.
- 전체화면에서는 카메라 종류와 FPS를 포함한 영상 오버레이도 표시하지 않는다.
- 빠른 작업 그룹은 유지하지 않고, 설정 진입은 녹화/재생 탭에 영향을 받지 않는 상시 톱니바퀴 버튼으로 제공한다.
- 이벤트 로그 진입점은 메인 화면에서 제거하고 설정창의 전용 페이지로 통합한다.
