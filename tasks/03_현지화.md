# 작업명

현지화언어구현

## 목표

* 모든 메뉴와 설명을 한국어(KOR) / 영어(ENG)로 전환할 수 있도록 한다.
* 언어 선택은 메뉴바에서 제공한다.
* 선택한 언어는 앱 재시작 후에도 유지되도록 설정에 저장한다.

## 배경

* 현재 WinForms 화면의 버튼, 라벨, 상태바, 메시지박스, 설정창, 트레이 메뉴 문자열이 코드와 Designer 파일에 흩어져 있다.
* 현재 일반적인 메뉴바가 거의 없으므로, 언어 선택용 메뉴바 추가가 필요하다.
* `Designer.cs` 직접 수정은 최소화하고, 가능하면 코드에서 현지화 적용 메서드로 텍스트를 갱신한다.

## 관련 파일

* `DFBlackbox/Forms/MainForm.cs`
* `DFBlackbox/Forms/MainForm.Designer.cs`
* `DFBlackbox/Forms/SettingsForm.cs`
* `DFBlackbox/Forms/EventListForm.cs`
* `DFBlackbox/Forms/CameraSettingsForm.cs`
* `DFBlackbox/Forms/PlaybackControl.cs`
* `DFBlackbox/Utils/SettingsManager.cs`
* `DFBlackbox/Models` 설정 관련 파일
* 필요 시 새 현지화 전용 클래스 또는 리소스 파일

## 변경 금지

* 기존 기능 동작을 바꾸지 않는다.
* 기존 설정 파일 형식은 하위 호환되도록 유지한다.
* DB, API, 녹화/재생 알고리즘, 카메라 연결 로직은 현지화 목적 외에는 수정하지 않는다.
* 보안 정보, 비밀번호, 토큰, 전체 RTSP URL을 새 로그나 문구에 포함하지 않는다.

## 요구사항

* 메뉴바에서 언어를 선택할 수 있어야 한다.
* 언어 메뉴에는 한국어(KOR), 영어(ENG)를 제공한다.
* 언어를 변경하면 현재 열려 있는 메인 화면의 버튼, 라벨, 상태바, 트레이 메뉴 텍스트가 즉시 갱신되어야 한다.
* 설정창, 최근 이벤트창, 카메라 속성창 등 별도 Form도 열릴 때 현재 언어로 표시되어야 한다.
* 상태바의 실시간 문구도 번역 대상이다.
  예: `녹화: 켜짐`, `감지: 대기`, `오류: 없음`
* 메시지박스 문구도 번역 대상이다.
* 디버그/알고리즘 내부 문구도 사용자가 볼 수 있는 경우 번역 대상이다.
  예: `baseline missing`, `algorithm off`
* 선택한 언어는 `settings.json`에 저장하고, 기존 설정 파일에 언어 값이 없어도 기본값 KOR로 정상 동작해야 한다.
* 애매한 문구나 번역 범위가 있으면 구현 전에 사용자에게 확인한다.

## 권장 구현 방향

* `Localization` 전용 클래스 또는 유사한 구조를 만든다.
* 화면 코드는 문자열 리터럴을 직접 박기보다 키 기반으로 텍스트를 가져오도록 정리한다.
* 각 Form에는 가능하면 `ApplyLocalization()` 형태의 메서드를 두어 언어 변경 시 텍스트를 다시 적용한다.
* `Designer.cs`에 있는 초기 텍스트는 유지하되, 런타임에서 현지화 텍스트로 덮어쓰는 방식을 우선 검토한다.

## 완료 조건

* 컴파일 에러 없이 동작할 것.
* 메뉴바에서 KOR/ENG를 전환할 수 있을 것.
* 언어 선택 후 앱을 재시작해도 선택 언어가 유지될 것.
* 메인 화면, 설정창, 최근 이벤트창, 카메라 속성창의 주요 사용자 표시 문구가 선택 언어로 표시될 것.
* 기존 녹화/재생/카메라 연결 동작이 회귀하지 않을 것.

## 테스트 방법

* `dotnet build DFBlackbox\DFBlackbox.csproj -o .buildcheck`
* 앱 실행 후 메뉴바에서 ENG 선택
* 메인 화면 버튼, 상태바, 트레이 메뉴, 메시지박스 문구 확인
* 설정창, 최근 이벤트창, 카메라 속성창을 열어 영어 표시 확인
* 앱 종료 후 재실행하여 ENG 선택 유지 확인
* KOR로 다시 변경 후 한국어 표시 확인
