# 작업명

DFBlackbox 메인/설정 화면 상태 중심 UI 재구성

## 목표

* 현재 적용된 Krypton Toolkit 기반 UI를 유지하면서 메인 화면과 설정 화면을 새 디자인 시안에 맞게 재구성한다.
* 메인 창은 클라이언트 영역 `1440 × 760px`, 설정 창은 외곽 크기 `1000 × 840px` 기준으로 설계한다.
* 메인 화면에서는 영상 표시 영역을 최대한 크게 확보하고, 오른쪽에는 핵심 상태와 주요 조작만 배치한다.
* 현재처럼 이벤트는 상시 목록으로 표시하지 않고 `이벤트 로그` 버튼을 통해 기존 `EventListForm`에서 확인하도록 한다.
* 카메라 연결, 녹화, 파일 재생, 감지, 전체화면, 설정 저장 등 기존 기능 동작은 변경하지 않는다.
* 이번 작업은 UI 배치와 시각적 정보 계층 개선을 목적으로 하며, 카메라/녹화/재생 엔진의 구조 변경은 포함하지 않는다.

## 배경

* 현재 메인 화면은 영상 표시 영역과 오른쪽 버튼 패널로 구성되어 있으나, 버튼들이 동일한 강조 수준으로 표시되어 현재 상태와 작업 우선순위를 빠르게 파악하기 어렵다.
* 하단 상태 정보가 한 줄 문자열로 연결되어 있어 카메라, RTSP, FPS, 녹화, 저장소, 오류 상태를 한눈에 확인하기 어렵다.
* 현재 메인 화면에는 최근 이벤트 상시 목록이 없고 `최근 이벤트` 버튼만 있으므로, 목록 제거 작업이 아니라 버튼 명칭과 별도 창 접근 흐름을 정돈하는 작업으로 본다.
* 설정 화면은 카메라, 녹화, 저장소, 시작 동작 관련 항목이 한 페이지에 혼재되어 있어 정보 탐색과 정렬이 어렵다.
* 현재 설정 화면에는 감지와 오버레이/재생 관련 설정도 있으므로 새 탐색 구조에서 누락하지 않는다.
* 기존 Krypton Toolkit 테마와 컨트롤 교체는 완료된 상태이므로, 이번 작업에서는 컨트롤 라이브러리 재도입보다 레이아웃과 상태 표현 개선에 집중한다.
* 기존 UI 개선 작업의 원칙인 기능 보존, 영상 표시 구조 유지, 단계적 변경 방식을 그대로 따른다.

### 확정 구현 기준

* 디자인 시안의 색상, 카드 구성, 버튼 계층, 설정 탐색 구조를 가능한 한 가깝게 따른다.
* 기존 Krypton 창 프레임을 유지하고 바로 아래에 네이비 상태 헤더를 배치한다. 사용자 정의 창 드래그/최대화/닫기 처리는 새로 만들지 않는다.
* 라이브 화면에는 실제로 제공되는 FPS만 표시한다. 실시간 수신 비트레이트와 네트워크 정상 여부는 계산 근거가 없으므로 표시하지 않는다.
* 녹화 비트레이트가 필요하면 `녹화 설정` 값임을 명확히 표시하며 라이브 스트림 비트레이트처럼 표현하지 않는다.
* 시안의 `일시 정지` 버튼은 제거한다.
* 시안의 `긴급 녹화`는 새 기능으로 만들지 않고 기존 `수동 녹화` 동작과 명칭을 유지한다.
* 시안의 `디스크 관리`는 기존 `저장 폴더 열기`로 대체한다.
* 이벤트 읽음 상태가 없으므로 이벤트 배지를 제거하고 기존 `EventListForm`만 연결한다.
* 카메라 이름 모델이 없으므로 영상 오버레이에는 IP 카메라 또는 USB 카메라 장치 번호를 표시한다.
* 실제 녹화 경로는 현재 `AppPaths.RecVideos`를 표시한다. 이번 작업에서는 저장 경로 선택 UI와 녹화 경로 정책을 변경하지 않는다.
* 설정 화면의 저장 위치는 현재 실제 녹화 경로를 읽기 전용으로 안내하고, 동작하지 않는 찾아보기 입력란은 만들지 않는다.
* 프로그램 정보는 현재 실행 버전과 기본 정보만 표시하며 새로운 업데이트/라이선스 관리 기능은 추가하지 않는다.

## 변경 금지

* 영상 표시 영역의 `PictureBox`, 프레임 렌더링, OpenCV 처리 구조를 UI 개선 목적으로 변경하지 않는다.
* 카메라 연결, RTSP 연결, USB 카메라 열기, 녹화 시작/중지, 감지, 파일 재생 로직을 변경하지 않는다.
* 기존 이벤트 핸들러의 의미를 바꾸지 않는다.
* 버튼 명칭을 변경하더라도 기존 동작과 연결된 이벤트는 그대로 유지한다.
* 파일 선택 재생과 드래그드롭 재생의 공통 흐름을 분리하지 않는다.
* 설정 파일 스키마와 기존 설정 키 이름을 변경하지 않는다.
* 녹화 파일 저장 규칙, 파일명 형식, 보관 및 정리 로직을 변경하지 않는다.
* 전체 RTSP URL, 인증 정보, 사용자 로컬 경로를 로그나 문서에 노출하지 않는다.
* UI 작업과 함께 성능 최적화, 스레드 구조 변경, 카메라 재연결 알고리즘 변경을 진행하지 않는다.
* `Designer.cs` 직접 수정은 필요한 범위로 제한하며, Visual Studio 디자이너가 다시 열리는지 확인한다.
* 새로운 외부 UI 프레임워크를 추가하지 않는다.
* 최근 이벤트 목록을 메인 화면에 다시 상시 표시하지 않는다.

## 요구사항

### 공통

* Krypton Toolkit과 기존 `UiTheme` 구조를 유지한다.
* 밝은 테마를 기반으로 하며, 네이비/인디고 계열을 주요 강조색으로 사용한다.
* 상태 색상은 제한적으로 사용한다.

  * 정상: 녹색
  * 녹화 중/심각 오류: 빨간색
  * 대기/주의: 노란색 또는 주황색
  * 정보/처리 중: 청록색 또는 파란색
* 상태는 색상만으로 표현하지 않고 아이콘과 텍스트를 함께 표시한다.
* 상태 정보의 역할을 상단, 상세 카드, 하단 상태바로 구분하고 같은 내용을 세 영역에 반복하지 않는다.
* 모든 버튼, 라벨, 입력 필드는 한국어와 영어에서 텍스트가 잘리지 않아야 한다.
* 100% 배율과 125% 배율에서 주요 레이아웃이 깨지지 않아야 한다.
* 폼 크기 축소 시 최소 크기 이하로 줄어들지 않도록 `MinimumSize`를 지정한다.

### 메인 창 크기

* 기준 크기: 클라이언트 영역 `1440 × 760px`
* 최소 크기 권장값: `1280 × 680px`
* 폼 시작 위치는 기존 정책을 유지하되, 화면을 벗어나지 않도록 한다.
* 상단 제목 표시 영역, 중앙 콘텐츠 영역, 하단 상태바의 3단 구조로 구성한다.

### 설정 창 크기

* 기준 크기: 현재 코드와 동일한 외곽 크기 `1000 × 840px`
* 최소 크기 권장값: `900 × 720px`
* 최소 크기까지 축소 가능한 창으로 변경할 경우 `FixedDialog` 정책도 함께 조정하고, 고정 대화상자를 유지한다면 최소 크기 요구 대신 내부 페이지 스크롤을 보장한다.
* 설정 항목이 세로로 길어질 경우 전체 폼을 늘리기보다 각 설정 페이지 내부 스크롤을 사용한다.
* 하단에는 고정된 작업 버튼 영역을 둔다.

## 메인 화면 레이아웃

### 전체 구조

```text
MainForm : KryptonForm
├─ TopHeaderPanel
│  ├─ 앱 이름
│  ├─ 카메라 연결 상태
│  ├─ 녹화 상태 및 경과 시간
│  └─ 저장 위치 및 여유 공간
├─ MainContentPanel
│  ├─ VideoArea
│  │  ├─ 기존 영상 표시 컨트롤
│  │  ├─ 영상 상태 오버레이
│  │  ├─ 촬영 시각 오버레이
│  │  └─ REC / FPS / 비트레이트 오버레이
│  └─ RightControlPanel
│     ├─ CameraStatusCard
│     ├─ RecordingStatusCard
│     ├─ StorageStatusCard
│     └─ QuickActionPanel
└─ BottomStatusBar
   ├─ 카메라
   ├─ RTSP
   ├─ FPS
   ├─ 비트레이트
   ├─ 감지
   ├─ 녹화
   ├─ 디스크
   └─ 오류
```

### 메인 화면 폭 배분

* 기준 크기에서 오른쪽 전체 조작 영역은 약 `420px`, 허용 범위는 `400\~460px`로 한다.
* 영상 영역은 남은 폭을 모두 사용해 약 `68\~72%`를 확보하고, 현재보다 좁아지지 않도록 한다.
* 상태 카드와 빠른 작업 버튼을 별도 세로 열로 나누지 않고 하나의 오른쪽 패널 안에 배치한다.
* 최근 이벤트 상시 목록은 배치하지 않는다.
* 현재 존재하지 않는 이벤트 목록 공간을 회수한다는 전제는 사용하지 않는다.
* 영상 영역은 가능한 한 넓게 유지하고, 오른쪽 패널은 필요한 정보만 간결하게 표시한다.

### 상단 헤더

* 왼쪽에 `DFBlackbox` 앱 이름을 표시한다.
* Krypton/Windows 창 프레임과 창 제어 버튼을 새로 직접 구현하지 않고, 헤더는 클라이언트 영역 안의 상태 표시 패널로 구성한다.
* 중앙 또는 오른쪽에 다음 핵심 상태를 표시한다.

  * `● 카메라 연결됨`
  * `● REC 00:15:32`
  * `D:\\\\\\\\Recordings (128.4 GB 여유)`
* 녹화 중이 아닐 때는 `REC` 대신 `녹화 대기` 또는 `녹화 꺼짐`을 표시한다.
* 저장소 정보를 확인할 수 없으면 `저장소 확인 안 됨`으로 표시한다.
* 상단 상태 표시는 기존 상태 데이터를 읽어 표시만 하며, 새 상태 계산 로직을 만들지 않는다.
* 상단에는 카메라 연결, 녹화 경과 시간, 디스크 여유 공간처럼 즉시 판단해야 하는 정보만 둔다.

### 영상 영역

* 기존 영상 출력 컨트롤과 렌더링 구조를 유지한다.
* 영상이 없을 때 검은 화면만 표시하지 않고 중앙 안내 오버레이를 표시한다.

상태별 예시:

```text
카메라가 연결되지 않았습니다.
오른쪽의 \\\\\\\[카메라 연결] 버튼을 누르세요.
```

```text
카메라에 연결하는 중...
192.168.10.100
```

```text
영상을 불러올 수 없습니다.
카메라와 네트워크 상태를 확인하세요.
```

* 라이브 영상 표시 중에는 오버레이를 숨긴다.
* 녹화 중에는 영상 우측 상단에 다음과 같은 표시를 추가한다.

```text
● REC 00:15:32
```

* 영상 좌측 하단에는 카메라 이름 또는 `CAM 01`을 표시할 수 있다.
* 영상 우측 하단에는 기존 데이터가 있을 때만 다음을 표시한다.

```text
FPS: 30 | 1.2 Mbps
```

* 오버레이는 영상 렌더링 이벤트나 프레임 버퍼 구조를 변경하지 않고 별도 라벨/패널로 구현한다.
* 전체화면 진입 시 기존 정책에 따라 오버레이 표시 여부를 유지한다.
* 파일 재생 중에는 기존 `PlaybackControl`을 영상 하단에 동적으로 표시하며, 라이브 화면의 상시 높이를 차지하지 않는다.
* 전체화면 진입점은 영상 영역의 작은 보조 버튼 또는 기존 단축키로 유지한다.

### 오른쪽 카메라 상태 카드

표시 항목:

```text
카메라

연결 상태    ● 연결됨
카메라 주소  192.168.10.100
스트림 상태  정상 (30 FPS, 320×240)

\\\\\\\[연결 해제] \\\\\\\[설정]
\\\\\\\[영상 중지]
```

* 연결 전에는 `카메라 연결` 버튼을 가장 눈에 띄는 주요 버튼으로 표시한다.
* 연결 후에는 `연결 해제`, `설정`, `영상 시작/중지`를 상황에 맞게 활성화한다.
* 기존 `카메라 열기` 문구는 가능하면 `영상 시작` 또는 `스트림 시작`으로 변경한다.
* 기존 `카메라 닫기` 문구는 가능하면 `영상 중지` 또는 `스트림 중지`로 변경한다.
* 명칭 변경 시 한국어/영어 현지화 항목을 함께 추가한다.
* 기능과 이벤트 핸들러는 기존 연결을 유지한다.

### 오른쪽 녹화 상태 카드

표시 항목:

```text
녹화

상태       ● 녹화 중
현재 파일  20260715\\\\\\\_160200.mp4
경과 시간  00:15:32
파일 크기  128.6 MB
저장 위치  D:\\\\\\\\Recordings\\\\\\\\

\\\\\\\[녹화 중지]
```

* `녹화 시작`은 현재 녹화 중이 아닐 때 가장 중요한 주요 버튼으로 표시한다.
* `녹화 중지`는 녹화 중일 때 빨간색 위험 작업 버튼으로 표시한다.
* 시안의 일시 정지 버튼은 구현하지 않는다. 녹화 일시 정지는 현재 지원하지 않는 기능이다.
* 현재 파일명과 파일 크기는 기존 서비스에서 안전하게 얻을 수 있을 때만 표시하며, 이를 위해 새 폴링 스레드나 파일 감시 구조를 만들지 않는다.
* 파일 경로 전체가 너무 길면 폴더명 또는 축약 경로만 표시하고 ToolTip에서 전체 경로를 확인할 수 있게 한다.
* `폴더 열기` 버튼은 보조 버튼으로 배치한다.

### 오른쪽 저장소 상태 카드

표시 항목:

```text
저장소

드라이브  D:\\\\\\\\ (128.4 GB 여유)
사용량    371.6 GB / 500 GB (25%)

\\\\\\\[저장 폴더 열기]
```

* 저장소 사용량을 간단한 진행 표시줄로 표현한다.
* 디스크 임계값에 따라 상태색을 변경한다.

  * 정상 범위: 녹색 또는 기본 강조색
  * 경고 범위: 노란색
  * 녹화 중지 임계값 도달: 빨간색
* 임계값 판단은 기존 설정값과 기존 디스크 검사 결과를 사용한다.
* 새 저장소 계산 스레드를 만들지 않는다.
* 시안의 `디스크 관리` 버튼은 추가하지 않고 기존 `저장소`/ `저장 폴더 열기` 동작을 재사용한다.

### 오른쪽 빠른 작업 패널

버튼 권장 순서:

```text
카메라 연결
카메라 설정
녹화 설정
저장소 설정
이벤트 로그
프로그램 설정
영상 불러오기
```

* `카메라 설정`, `녹화 설정`, `저장소 설정`, `프로그램 설정`은 동일한 `SettingsForm`을 열되 가능하면 대응 페이지를 초기 선택한다.
* `카메라 설정`과 USB 전용 카메라 속성 창을 문구와 아이콘으로 명확히 구분한다.
* `이벤트 로그` 버튼을 누르면 기존 `EventListForm`을 일반 도구 창으로 연다.
* 미확인 이벤트 상태를 관리하는 기존 데이터가 없으므로 이벤트 배지는 이번 범위에서 추가하지 않는다.
* `최근 이벤트` 목록이나 카드 영역은 메인 화면에 추가하지 않는다.
* 기존 수동 녹화를 새로운 `긴급 녹화` 기능처럼 표현하지 않는다. 메인 녹화 카드의 `녹화 시작`이 기존 수동 녹화 동작을 그대로 호출한다.
* 프로그램 정보, 디스크 관리, 녹화 일시 정지처럼 현재 구현되지 않은 기능은 시안에 있다는 이유만으로 추가하지 않는다.
* `저장`, `기본값` 버튼은 메인 화면에서 제거한다.
* 설정 저장과 기본값 복원은 설정 창에만 둔다.
* `감시 시작/중지`, `차이 기준 저장`, 파일 재생, 전체화면의 진입점을 제거하지 않는다. 자주 쓰지 않는 감지 동작은 감지 카드나 보조 작업 영역에 배치한다.

### 하단 상태바

현재 한 줄 문자열을 항목별 상태로 분리한다.

권장 구조:

```text
RTSP: 정상
FPS: 30
비트레이트: 1.2 Mbps
감지: 대기
마지막 프레임: 0.2초 전
오류: 없음
```

* 카메라 연결, 녹화, 디스크 상태는 상단과 상세 카드에서 보여 주므로 하단 상태바에서는 반복하지 않는다.
* 각 항목은 구분선으로 나눈다.
* 상태가 변경된 항목만 텍스트와 아이콘을 갱신한다.
* 오류 발생 시 오류 항목을 빨간색으로 표시한다.
* 모든 정보가 들어가지 않으면 우선순위가 낮은 항목은 ToolTip 또는 상세 상태 창으로 이동한다.
* 상태바 우측 끝에는 앱 버전을 표시할 수 있다.

## 이벤트 로그 창

### 기본 원칙

* 최근 이벤트는 메인 화면에 상시 표시하지 않는다.
* `이벤트 로그` 버튼을 눌렀을 때 기존 `EventListForm`을 모달이 아닌 일반 도구 창으로 연다.
* 기존 이벤트 데이터와 기록 방식을 재사용한다.
* 이벤트 로그 창을 닫아도 카메라 연결과 녹화가 중단되지 않아야 한다.

### 권장 구성

```text
EventListForm : Form 또는 KryptonForm
├─ 상단 필터
│  ├─ 기간
│  ├─ 수준
│  ├─ 검색어
│  └─ 새로고침
├─ 이벤트 목록
│  ├─ 시각
│  ├─ 수준
│  ├─ 구분
│  └─ 내용
└─ 하단 작업
   ├─ 선택 항목 복사
   ├─ 로그 폴더 열기
   └─ 닫기
```

* 최초 구현은 단순 목록과 닫기 버튼만으로 시작해도 된다.
* 기존 `EventListForm`의 공개 클래스명과 이벤트 연결을 유지하고 새 폼을 중복 추가하지 않는다.
* 이벤트 수준은 정보, 경고, 오류로 구분한다.
* 개인정보, 인증 정보, 전체 RTSP URL은 목록에 표시하지 않는다.

## 설정 화면 레이아웃

### 전체 구조

설정 창은 왼쪽 탐색 메뉴와 오른쪽 설정 페이지로 구성한다.

```text
SettingsForm : KryptonForm
├─ LeftNavigation
│  ├─ 카메라
│  ├─ 녹화
│  ├─ 감지
│  ├─ 저장소
│  ├─ 화면 및 재생
│  ├─ 일반 및 시작
│  ├─ 고급
│  └─ 정보
├─ ContentPanel
│  └─ 선택된 설정 페이지
└─ BottomActionPanel
   ├─ 기본값 복원
   ├─ 취소
   ├─ 적용
   └─ 확인
```

* `KryptonNavigator`를 사용할 수 있으나, 현재 패키지 범위를 늘려야 한다면 기본 패널 전환 방식으로 구현해도 된다.
* 새로운 Krypton 패키지는 실제 필요성이 확인될 때만 추가한다.
* 왼쪽 메뉴 폭은 약 `170\\\\\\\~190px`로 한다.
* 오른쪽 내용 영역은 나머지 폭을 사용한다.
* 하단 작업 버튼 영역은 페이지 전환과 무관하게 고정한다.
* 각 페이지는 독립적인 내부 스크롤을 사용하고 페이지 전환 시 아직 저장하지 않은 입력값을 유지한다.

### 카메라 페이지

카메라 종류:

```text
(●) IP 카메라   ( ) USB 카메라
```

IP 카메라 선택 시 표시:

```text
IP 주소       \\\\\\\[192.168.10.100]
RTSP 포트     \\\\\\\[554]
HTTP 포트     \\\\\\\[80]

\\\\\\\[카메라 찾기]

검색 결과     \\\\\\\[DF Camera (ONVIF) - 192.168.10.100 ▼]

스트림 경로   \\\\\\\[/stream1]
RTSP URL      \\\\\\\[rtsp://192.168.10.100:554/stream1]

\\\\\\\[연결 테스트]

▶ 고급 RTSP 설정
```

스트림 설정:

```text
해상도       \\\\\\\[320×240 ▼]
FPS          \\\\\\\[30 ▼]
```

* USB 카메라 선택 시 IP/RTSP 관련 영역은 숨긴다.
* IP 카메라 선택 시 USB 장치 관련 영역은 숨긴다.
* 사용하지 않는 컨트롤을 단순 비활성화만 하지 말고 가능하면 레이아웃에서 숨긴다.
* `웹 설정`은 `카메라 관리 페이지 열기`로 명칭을 변경한다.
* 수동 RTSP URL은 `고급 RTSP 설정`을 펼쳤을 때만 표시한다.
* 연결 테스트는 기존 연결 로직을 재사용하되, 실제 라이브 스트림 시작과 분리할 수 없는 구조라면 새 기능을 억지로 만들지 않는다.
* `스트림 선택`, 코덱, 비트레이트처럼 현재 설정 키나 신뢰할 수 있는 런타임 데이터가 없는 항목은 표시하지 않는다.

### 녹화 페이지

권장 항목:

```text
녹화 방식
(●) 수동
( ) 전체
( ) 감지

녹화 파일 분할 시간
\\\\\\\[10] 분마다 새 파일 생성

녹화 비트 전송률
\\\\\\\[낮음 (320 kbps) ▼]

\\\\\\\[ ] 앱 시작 시 전체 녹화 시작
```

* 현재 `전체 녹화 간격(분)`이 파일 분할 시간 의미라면 `녹화 파일 분할 시간`으로 명칭을 변경한다.
* 실제 의미가 다르면 기존 로직을 확인해 정확한 명칭을 사용한다.
* 녹화 모드 선택은 기존 상태 전환과 동일하게 동작해야 한다.
* 지원하지 않는 일시정지, 예약 녹화 등은 추가하지 않는다.

### 감지 페이지

현재 설정 항목을 다음과 같이 이동한다.

```text
ROI 차이 임계값
녹화 정지 대기 시간
사전 녹화 버퍼 시간
사전 버퍼 최대 메모리
```

* 현재 `DetectionSettings` 및 설정 컨트롤과의 매핑을 그대로 유지한다.
* 감시 시작/중지와 차이 기준 저장은 실행 동작이므로 설정값과 구분하고 메인 화면의 감지 보조 작업으로 유지한다.
* 모델에만 존재하고 현재 설정 화면에서 편집하지 않는 감지 값을 이번 UI 작업을 이유로 새로 노출하지 않는다.

### 저장소 페이지

권장 항목:

```text
녹화 파일 저장 폴더
\\\\\\\[D:\\\\\\\\DFBlackbox\\\\\\\\Recordings] \\\\\\\[찾아보기]

현재 여유 공간
128.4 GB

녹화 중지 기준
\\\\\\\[85] %

녹화 재개 기준
\\\\\\\[70] %

녹화 파일 보관 기간
\\\\\\\[14] 일

자동 정리 시간
\\\\\\\[03:00]

\\\\\\\[✓] 앱 시작 시 정리
```

* `디스크 정지 %`는 `녹화 중지 기준`으로 변경한다.
* `디스크 재개 %`는 `녹화 재개 기준`으로 변경한다.
* 재개 기준은 중지 기준보다 낮아야 한다.
* 잘못된 값 입력 시 저장 전에 오류 메시지를 표시한다.
* 저장 위치가 유효하지 않거나 쓰기 권한이 없으면 명확한 오류를 표시한다.
* 예상 녹화 가능 시간은 기존 정보만으로 안전하게 계산할 수 있을 때만 표시한다.

### 화면 및 재생 페이지

현재 설정 항목을 다음과 같이 이동한다.

```text
ROI 표시
디버그 텍스트
재생 ROI 외곽선
재생 차이 메시지
재생 추적 후보
재생 최적화: 균형 / 재생 우선 / 추적 우선
```

* 현재 `OverlaySettings`와 컨트롤 매핑을 그대로 유지한다.
* 라이브 오버레이와 파일 재생 오버레이 설정을 시각적으로 구분한다.
* 기존 재생 최적화 세 모드의 의미와 저장 값은 변경하지 않는다.

### 일반 및 시작 페이지

권장 항목:

```text
언어  \\\\\\\[한국어 ▼]
\\\\\\\[✓] 시작 시 저장소 정리
\\\\\\\[ ] 트레이에서 시작
```

* 현재 구현되어 있는 기능만 표시한다.
* 기존 메인 메뉴의 KOR/ENG 언어 전환을 이 페이지로 이동하거나, 메인 메뉴를 유지한다면 중복 컨트롤을 만들지 않는다.
* Windows 시작 시 실행과 최소화 시 트레이 이동 설정은 현재 설정 키가 없으므로 추가하지 않는다.
* 기존 `트레이에서 시작` 설정과 동작은 유지한다.

### 고급 페이지

권장 항목:

```text
재연결 간격
프레임 없음 제한 시간
수동 RTSP URL
```

* 기존에 설정값이 존재하는 항목만 이동한다.
* 현재 `ReconnectDelaySeconds`, `NoFrameTimeoutSeconds`, 수동 RTSP URL 이외의 로그 수준이나 디코더 옵션은 새로 추가하지 않는다.
* 기존 설정 파일 스키마를 바꾸지 않는다.
* 일반 사용자가 자주 변경하지 않는 항목만 배치한다.

### 정보 페이지

권장 항목:

```text
DFBlackbox
버전
Krypton Toolkit 라이선스
오픈소스 고지
로그 폴더 열기
```

* 기존 프로그램 정보 창이 있다면 중복 구현하지 않고 연결한다.

### 설정 하단 버튼

권장 배치:

```text
\\\\\\\[기본값 복원]                    \\\\\\\[취소] \\\\\\\[적용] \\\\\\\[확인]
```

* `기본값 복원`은 좌측에 둔다.
* `확인`, `적용`, `취소`는 우측에 둔다.
* 확인: 저장 후 닫기
* 적용: 저장 후 창 유지
* 취소: 저장하지 않고 닫기
* 기본값 복원 시 즉시 저장하지 말고 화면 값만 변경한 후 사용자가 확인 또는 적용을 눌러야 저장되도록 한다.
* 기존 동작이 다르다면 기존 정책을 우선하며 문서에 차이를 기록한다.

* 상단 헤더: 진한 네이비
* 주요 버튼: 파란색 또는 인디고
* 보조 버튼: 흰색/연회색
* 녹화 상태: 빨간색
* 정상 상태: 녹색
* 경고 상태: 노란색 또는 주황색
* 정보 상태: 청록색 또는 파란색
* 비활성 상태: 중립 회색

권장 예시:

```text
상단 헤더        #081B33
주요 액션        #007BFF
정상             #28A745
경고             #FFC107
오류/녹화        #DC3545
정보             #17A2B8
기본 배경        #F5F7FA
카드 배경        #FFFFFF
테두리           #D8DEE8
```

* 색상 값은 기존 `UiTheme` 구조에서 관리한다.
* 컨트롤 코드에 색상을 반복 하드코딩하지 않는다.

### 글꼴과 크기

* 기본 폰트는 기존 앱 폰트를 유지한다.
* 본문: 약 `9\\\\\\\~10pt`
* 섹션 제목: 약 `11\\\\\\\~12pt`, Bold
* 핵심 상태: 약 `11\\\\\\\~13pt`, Bold
* 주요 버튼 높이: `38\\\\\\\~44px`
* 일반 버튼 높이: `32\\\\\\\~36px`
* 입력 컨트롤 높이는 한 페이지에서 통일한다.
* 한국어/영어 전환 시 텍스트 잘림이 없도록 자동 크기와 최소 폭을 함께 조정한다.

### 여백

* 카드 내부 여백: `12\\\\\\\~16px`
* 카드 사이 간격: `8\\\\\\\~12px`
* 설정 페이지의 행 간격: `10\\\\\\\~14px`
* 설정 그룹 간 간격: `18\\\\\\\~24px`
* 외곽 패널 여백: `12\\\\\\\~16px`

### 버튼 계층

* 주요 버튼

  * 카메라 연결
  * 영상 시작
  * 녹화 시작
  * 확인/저장
* 보조 버튼

  * 설정
  * 폴더 열기
  * 카메라 찾기
  * 이벤트 로그
* 위험 버튼

  * 녹화 중지
  * 연결 해제
  * 기본값 복원
* 모든 버튼을 같은 파란색 배경으로 표시하지 않는다.
* 상태에 따라 필요한 버튼만 활성화한다.
* 비활성 이유는 ToolTip으로 안내한다.

## 구현 순서

### 1차 작업: 메인 레이아웃 골격

* 메인 창 기준 크기와 최소 크기를 설정한다.
* 오른쪽 전체 조작 영역을 약 420px의 단일 열로 구성하고 영상 영역이 현재보다 좁아지지 않도록 한다.
* 현재 이벤트 상시 목록이 없음을 확인하고 해당 구조를 그대로 유지한다.
* 오른쪽 영역을 카메라, 녹화, 저장소, 빠른 작업 카드로 구성한다.
* 기존 버튼 이벤트를 새 버튼에 다시 연결한다.
* 영상 불러오기, 감시 시작/중지, 차이 기준 저장, 전체화면 진입점을 새 레이아웃에 보존한다.
* 영상 표시 컨트롤의 Dock/Anchor 구조가 깨지지 않는지 확인한다.

### 2차 작업: 메인 상태 표시

* 상단 헤더에 카메라, 녹화, 저장소 핵심 상태를 표시한다.
* 영상 없음/연결 중/오류 상태 오버레이를 추가한다.
* 녹화 중 REC 오버레이를 추가한다.
* 하단 상태 문자열을 항목별 상태 표시로 분리한다.
* 상단, 상세 카드, 하단 상태바의 역할을 나누고 카메라/녹화/디스크 상태를 중복 표시하지 않는다.
* 기존 상태 갱신 코드에서 UI 표시만 연결한다.

### 3차 작업: 이벤트 로그 분리

* 기존 `최근 이벤트` 버튼을 `이벤트 로그` 버튼으로 정리한다.
* 기존 `EventListForm`과 이벤트 데이터 연결을 그대로 재사용한다.
* 새 `EventLogForm`이나 미확인 이벤트 배지 데이터 구조를 추가하지 않는다.
* 이벤트 로그 창을 열고 닫아도 메인 폼 상태와 녹화가 영향을 받지 않는지 확인한다.

### 4차 작업: 설정 창 골격

* 설정 창 기준 크기와 최소 크기를 설정한다.
* 왼쪽 탐색 메뉴와 오른쪽 페이지 영역을 만든다.
* 기존 설정 컨트롤을 카메라, 녹화, 감지, 저장소, 화면 및 재생, 일반 및 시작, 고급 페이지로 이동한다.
* 설정값 읽기/쓰기 코드와 컨트롤 매핑은 유지한다.

### 5차 작업: 설정 페이지 정돈

* IP/USB 카메라 선택에 따라 관련 영역을 숨기고 표시한다.
* 라벨, 입력 컨트롤, 단위의 정렬을 통일한다.
* 용어를 사용자 중심으로 수정한다.
* 수동 RTSP 설정을 고급 영역으로 이동한다.
* KOR/ENG 언어 전환의 위치를 정리하되 기존 언어 저장과 적용 흐름을 유지한다.
* 디스크 임계값 검증을 추가한다.
* 하단 버튼 영역을 고정한다.

### 6차 작업: 테마와 현지화

* `UiTheme`에 헤더, 카드, 상태, 주요/보조/위험 버튼 스타일을 추가한다.
* 새 문구를 `Localization` 구조에 등록한다.
* 한국어와 영어에서 텍스트 잘림을 확인한다.
* 100%, 125% DPI에서 확인한다.

### 7차 작업: 기능 회귀 테스트

* 카메라 연결/해제
* 영상 시작/중지
* 녹화 시작/중지
* 수동/전체/감지 모드
* 파일 선택 재생
* 드래그드롭 재생
* 전체화면 진입/종료
* 설정 확인/적용/취소
* 저장소 정리
* 이벤트 로그 열기/닫기

## 완료 조건

* 메인 창이 클라이언트 영역 `1440 × 760px` 기준으로 의도한 레이아웃을 표시한다.
* 설정 창이 외곽 크기 `1000 × 840px` 기준으로 의도한 레이아웃을 표시한다.
* 메인 영상 영역이 기존 화면보다 넓어지고 핵심 시각 영역으로 유지된다.
* 메인 화면에 이벤트 상시 목록이 없는 현재 구조를 유지하고 `이벤트 로그` 버튼으로 기존 `EventListForm`에 접근할 수 있다.
* 카메라, 녹화, 저장소의 현재 상태를 메인 화면에서 한눈에 확인할 수 있다.
* 영상이 없을 때 상태 안내 오버레이가 표시된다.
* 녹화 중 REC 표시와 경과 시간이 표시된다.
* 메인 하단 상태바가 항목별로 구분된다.
* 상단, 상세 카드, 하단 상태바에 같은 상태 정보가 불필요하게 반복되지 않는다.
* 설정 화면이 카메라, 녹화, 감지, 저장소, 화면 및 재생, 일반 및 시작, 고급 항목으로 구분된다.
* IP/USB 카메라 선택에 따라 불필요한 입력 영역이 숨겨진다.
* 설정 저장, 적용, 취소가 기존과 동일하게 동작한다.
* 기존 카메라 연결, 녹화, 감지, 재생, 드래그드롭, 전체화면 기능이 유지된다.
* KOR/ENG 전환 시 텍스트가 잘리지 않는다.
* 빌드 경고와 오류가 새로 발생하지 않는다.
* UI 변경으로 FPS, 재생 속도, 녹화 안정성에 유의미한 저하가 없다.

### 후속 UI 보완 (2026-07-15)

* 오른쪽 작업 영역을 `녹화`와 `재생` 탭으로 분리했다. 녹화 탭에서는 영상 불러오기 버튼을 제거하고 기존 카메라/녹화/저장소/빠른 작업만 표시한다.

* KOR/ENG용 녹화/재생 탭 및 재생 파일 문구를 추가했다.
