# 작업명

동영상 파일 드래그 드롭 재생

## 목표

* 사용자가 영상 표시 영역에 동영상 파일을 드래그하면 재생 가능 여부를 시각적으로 알 수 있게 한다.
* 지원되는 동영상 파일을 드롭하면 기존 재생 기능으로 해당 파일을 바로 재생한다.
* 지원되지 않는 파일이나 복수 파일 드롭은 아무 동작 없이 거부 상태만 표시한다.

## 배경

* 기존에는 버튼으로 파일을 선택해 녹화 영상을 재생했다.
* 이번 작업은 같은 재생 기능을 드래그 드롭으로도 사용할 수 있게 하는 것이다.
* 기존 카메라, 녹화, 재생 흐름은 유지한다.

## 변경 금지

* 기존 녹화/재생/카메라 연결 알고리즘을 변경하지 않는다.
* DB, 설정 스키마, 배포 설정, 패키지 설정은 변경하지 않는다.
* 전체 RTSP URL, 로컬 사용자 경로, 인증 정보가 로그에 남지 않도록 한다.
* 드래그 드롭 지원을 위해 외부 라이브러리를 추가하지 않는다.

## 요구사항

* 지원 파일 확장자는 `.mp4`, `.avi`, `.mov`, `.mkv`, `.wmv`이다.
* 지원 파일을 드래그하면 테두리와 오버레이 문구로 드롭 가능 상태를 표시한다.
* 드롭 가능 문구는 `드래그 앤 드롭으로 재생하기`이다.
* 미지원 파일 또는 복수 파일을 드래그하면 금지 커서와 거부 상태를 표시한다.
* 미지원 파일을 드롭해도 메시지박스 없이 아무 일도 일어나지 않는다.
* 녹화 중인 상태에서 지원 동영상 파일을 드롭하면 기존 재생 전환 흐름에 따라 카메라 연결을 끊고 재생으로 전환한다.
* 지원 파일 드롭 후에는 자동 재생을 시작한다.
* KOR/ENG 언어 선택 상태에 맞춰 새 사용자 표시 문구가 표시된다.

## 완료 조건

* 컴파일 에러 없이 빌드된다.
* 지원 동영상 파일 드래그 시 드롭 가능 효과가 보인다.
* 미지원 파일 또는 복수 파일 드래그 시 드롭 불가 효과가 보인다.
* 지원 동영상 파일 드롭 시 재생 모드로 진입하고 자동 재생된다.
* 미지원 파일 또는 복수 파일 드롭 시 기존 상태가 깨지지 않는다.
* 드래그 취소 또는 영역 이탈 시 시각 효과가 원래 상태로 복원된다.

## 테스트 방법

* `dotnet build DFBlackbox\DFBlackbox.csproj -o .buildcheck`
* 앱 실행 후 `.mp4` 파일을 영상 표시 영역으로 드래그하여 드롭 가능 효과를 확인한다.
* `.mp4` 파일을 드롭하여 재생 모드 진입과 자동 재생을 확인한다.
* `.txt` 또는 미지원 확장자 파일을 드래그하여 드롭 불가 효과와 무시 동작을 확인한다.
* 2개 이상의 파일을 동시에 드래그하여 드롭 불가 효과와 무시 동작을 확인한다.
* 드래그 중 영역 밖으로 이동하거나 취소했을 때 시각 효과가 복원되는지 확인한다.
* ENG 언어 선택 후 같은 동작을 반복하여 새 문구가 영어로 표시되는지 확인한다.

## 결과

* 영상 표시 영역에 동영상 파일 드래그 드롭 재생 기능을 추가했다.
* 지원 파일 드래그 시 테두리/오버레이 안내 문구를 표시하고, 드롭 후 자동 재생하도록 했다.
* 미지원 파일 또는 복수 파일은 금지 커서와 거부 오버레이로 표시하고 드롭 시 재생하지 않도록 했다.
* 새 드래그 드롭 안내 문구를 KOR/ENG 현지화에 연결했다.
* 드롭 후 안내 문구가 재생 화면에 남지 않도록 표시 방식을 보정했다.
* `dotnet build DFBlackbox\DFBlackbox.csproj -o .buildcheck` 빌드가 경고 0개, 오류 0개로 성공했다.
