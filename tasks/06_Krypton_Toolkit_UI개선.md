# 작업명

Krypton Toolkit 도입 및 UI 외관 개선

## 목표

* Krypton Toolkit을 도입해 DFBlackbox가 기본 Windows Forms 앱처럼 보이지 않도록 외관을 개선한다.
* 기능 개선이나 성능 개선은 이번 작업의 목표로 삼지 않는다.
* 기존 카메라 연결, 녹화, 감지, 파일 선택 재생, 드래그드롭 재생, 재생 최적화 동작에 영향을 주지 않는다.
* UI 변경은 설정창과 오른쪽 조작 패널처럼 위험도가 낮은 영역부터 단계적으로 적용한다.

## 배경

* 현재 프로젝트는 .NET 8 Windows Forms 기반이며 기본 WinForms 컨트롤 중심으로 구성되어 있다.
* DevExpress는 강력하지만 비용과 의존성 부담이 크므로, 우선 오픈소스 WinForms UI 라이브러리인 Krypton Toolkit을 검토한다.
* Krypton Toolkit은 버튼, 패널, 그룹, 입력 컨트롤, 탭, 테마/팔레트 적용에 강점이 있다.
* DFBlackbox의 핵심은 영상 표시와 녹화/재생 처리이므로 UI 라이브러리 변경이 성능 문제를 해결한다고 기대하지 않는다.
* 외관 개선은 사용자 조작 영역의 정돈감, 테마 일관성, 기본 Windows 앱 느낌 완화를 목표로 한다.

## 변경 금지

* 영상 표시 영역의 재생/렌더링 구조를 UI 개선 목적으로 변경하지 않는다.
* OpenCV 재생 루프, 프레임 버퍼, Bitmap/GDI 객체 소유권, FPS 보정 로직을 외관 변경과 함께 수정하지 않는다.
* 카메라 연결, 녹화, 감지 알고리즘의 의미를 바꾸지 않는다.
* 파일 선택 재생과 드래그드롭 재생의 공통 재생 흐름을 분리하지 않는다.
* 설정 파일 스키마를 외관 개선 목적으로 변경하지 않는다.
* 전체 RTSP URL, 인증 정보, 사용자 로컬 경로가 로그나 문서에 남지 않도록 한다.
* WinForms `Designer.cs` 직접 수정은 최소화한다. 필요한 경우 변경 범위와 레이아웃 영향도를 확인한다.
* 기능 변경, 성능 최적화, 구조 리팩토링을 이번 작업에 끼워 넣지 않는다.

## 요구사항

* NuGet 패키지로 Krypton Toolkit을 도입한다.
* 패키지 도입 후 빌드와 게시 설정에 불필요한 변화가 생기지 않도록 확인한다.
* 앱 전체에 기본 Krypton 팔레트 또는 커스텀 팔레트를 적용한다.
* 색상은 영상 감시/녹화 도구에 맞게 차분하고 작업 중심적인 톤으로 잡는다.
* 기본 Windows Forms 회색 UI처럼 보이지 않도록 버튼, 그룹, 패널, 탭, 입력 컨트롤의 시각 스타일을 통일한다.
* 설정창의 입력 컨트롤과 그룹 영역을 우선 변경한다.
* 메인 화면의 오른쪽 조작 패널, 녹화 모드 선택, 상태성 버튼을 다음 우선순위로 변경한다.
* 영상 표시 영역은 기존 `PictureBox` 또는 현재 표시 구조를 유지한다.
* 재생 컨트롤러는 기존 동작을 유지하되, 외곽 패널/버튼 외관만 가능한 범위에서 개선한다.
* 새 사용자 표시 문구가 필요하면 기존 KOR/ENG 현지화 구조를 사용한다.
* 코드 주석과 작업 문서는 한국어로 작성한다.

## 권장 적용 범위

* 1차 적용
  * 프로젝트에 Krypton Toolkit 패키지를 추가한다.
  * 앱 시작 시 전역 팔레트를 설정한다.
  * `SettingsForm`의 버튼, 그룹, 입력 컨트롤을 Krypton 컨트롤로 교체한다.
  * 설정 저장/취소/카메라 찾기 버튼의 동작이 기존과 같은지 확인한다.
* 2차 적용
  * `MainForm` 오른쪽 조작 패널의 주요 버튼과 그룹 영역을 Krypton 컨트롤로 교체한다.
  * 녹화 모드 선택 UI가 기존 상태 전환과 동일하게 동작하는지 확인한다.
  * 전체화면 진입/종료 시 숨김 처리와 레이아웃이 깨지지 않는지 확인한다.
* 3차 적용
  * 재생 패널 외관을 앱 테마와 맞춘다.
  * 상태바 또는 상태 라벨의 색상/간격을 정리한다.
  * 필요하면 README에 Krypton Toolkit 도입 사실과 UI 의존성을 기록한다.

## 디자인 기준

* 기본 Windows Forms 컨트롤을 그대로 둔 느낌을 줄인다.
* 영상 영역이 주인공이 되도록 UI는 과하게 장식하지 않는다.
* 어두운 단색 UI로만 몰아가지 말고, 상태 색상과 강조 색상을 제한적으로 사용한다.
* 녹화, 감지, 재생 상태는 한눈에 구분되도록 색상과 텍스트 대비를 유지한다.
* 버튼과 입력 영역의 높이, 여백, 그룹 구분을 일관되게 정리한다.
* 작은 창 크기에서도 텍스트가 잘리지 않도록 확인한다.
* 한국어와 영어 UI 모두에서 버튼/라벨 텍스트가 컨트롤 안에 들어가는지 확인한다.

  * 선택 C: 전체 폼을 한 번에 교체한다.
* 기존 커스텀 재생 컨트롤 변경 여부 결정 필요.
  * 선택 B: 내부 버튼까지 Krypton 컨트롤로 교체한다.

## 완료 조건

* 컴파일 에러 없이 빌드된다.
* Krypton Toolkit 패키지가 프로젝트에 정상 참조된다.
* 앱 실행 시 기본 Windows Forms 회색 UI 느낌이 완화된다.
* 설정창의 주요 버튼, 입력 컨트롤, 그룹 영역이 일관된 스타일로 표시된다.
* 메인 화면의 조작 패널을 변경했다면 기존 버튼 동작이 그대로 유지된다.
* 카메라 연결, 카메라 열기, 녹화 시작/중지, 감지, 파일 선택 재생, 드래그드롭 재생이 기존처럼 동작한다.
* 전체화면 전환 시 오른쪽 패널, 상태바, 재생 패널 숨김/복원이 기존처럼 동작한다.
* KOR/ENG 언어 전환 후 새 UI 문구와 기존 문구가 잘리지 않는다.
* 영상 표시 영역의 프레임 갱신, 재생 속도, 오버레이 표시가 UI 변경 전과 같은 의미를 유지한다.
* 불필요한 성능 최적화나 기능 변경이 함께 섞이지 않는다.

## 테스트 방법

* `dotnet build DFBlackbox\DFBlackbox.csproj -o .buildcheck`
* 앱을 실행해 메인 화면이 기본 Windows Forms 앱처럼 보이지 않는지 확인한다.
* 설정창을 열고 각 입력 컨트롤, 콤보박스, 버튼, 그룹 영역의 외관과 정렬을 확인한다.
* 설정 저장과 취소가 기존처럼 동작하는지 확인한다.
* KOR/ENG 언어를 각각 선택해 텍스트 잘림과 레이아웃 깨짐을 확인한다.
* IP 카메라 또는 USB 카메라 연결 버튼 흐름이 기존처럼 동작하는지 확인한다.
* 수동/자동/전체 녹화 모드 버튼이 기존처럼 상태를 전환하는지 확인한다.
* 파일 선택으로 동영상을 불러와 재생 패널과 재생 컨트롤이 표시되는지 확인한다.
* 같은 동영상을 드래그드롭으로 불러와 파일 선택과 동일한 흐름으로 재생 모드에 진입하는지 확인한다.
* 전체화면 전환 후 UI 숨김/복원이 정상인지 확인한다.
* 게시 확인이 필요하면 `dotnet publish DFBlackbox\DFBlackbox.csproj -c Release -o .publishcheck`를 실행한다.

## 결과

* `Krypton.Toolkit` NuGet 패키지를 프로젝트에 추가했다.
* 앱 시작 시 Krypton 전역 팔레트 `Office2013White`를 적용해 기본 하늘색 톤이 과하게 드러나지 않도록 했다.
* `UiTheme` 유틸리티를 추가해 폼, 패널, 메뉴, 상태바의 공통 색상과 외관을 정리했다.
* `MainForm`과 `SettingsForm`을 `KryptonForm` 기반으로 전환했다.
* `MainForm`의 주요 버튼, 라디오 버튼, 체크박스, 텍스트박스, 콤보박스, 숫자 입력 컨트롤을 Krypton 컨트롤로 교체했다.
* `SettingsForm`의 주요 버튼, 라디오 버튼, 체크박스, 텍스트박스, 콤보박스, 숫자 입력 컨트롤을 Krypton 컨트롤로 교체했다.
* 메인 오른쪽 조작 패널, 상태바, 재생 패널에 차분한 밝은 톤을 적용했다.
* 버튼 상하 간격을 늘리고, 버튼/입력 컨트롤의 색상을 인디고 강조 계열로 조정했다.
* 버튼과 입력 컨트롤의 기본, 마우스오버, 눌림, 비활성 상태 색상을 `UiTheme`에서 명시해 상태 구분이 분명하게 보이도록 통일했다.
* 기본 버튼 색상을 진한 청색 계열로 변경하고, 마우스오버는 밝은 청색, 클릭은 더 어두운 청색으로 구분했다.
* 설정창 버튼에도 같은 버튼 테마가 적용되도록 하고, 버튼 내부 글씨 크기와 버튼 높이를 약간 키웠다.
* 설정창의 버튼 색상도 공통 파란 계통 버튼 테마로 통일하고, 주요 입력 라벨은 굵은 글씨로 표시하도록 조정했다.
* 설정창 버튼은 색상 적용이 명확하도록 플랫 버튼 테마를 직접 적용하고, 메인 창 버튼 글씨도 굵은 글씨로 통일했다.
* 설정창은 설정 로드와 카메라 타입별 활성/비활성 전환 이후 버튼 색상이 기본값으로 되돌아갈 수 있어, 표시 직후와 상태 변경 후 공통 테마를 재적용하도록 보강했다.
* 비활성 버튼은 파란 배경을 쓰지 않고 회색 계열로 표시해 버튼 글씨가 묻히지 않도록 했다.
* 입력 컨트롤의 활성 외곽선은 중립 색으로 낮춰 내부 입력 영역 선과 겹쳐 보이는 느낌을 줄였다.
* 설정창의 기준 폭과 주요 입력 행 너비를 넓혀 해상도, FPS, RTSP/HTTP, 저장소 설정 필드가 오른쪽에서 잘리거나 좁아지지 않도록 조정했다.
* 재생 컨트롤은 A번 원형 버튼형을 선택했다.
* 재생 컨트롤은 기존 이벤트/마우스 처리 구조를 유지하고, 타임라인과 내부 버튼의 그리기 색상을 인디고 테마 기본 톤에 맞게 조정했다.
* Krypton 체크박스/라디오버튼이 투명 배경을 지원하지 않아 실행 오류가 발생하지 않도록 불투명 배경색을 사용하도록 보정했다.
* Navigator, Ribbon, Docking 추가 패키지는 현재 화면 구조에 필요하지 않아 도입하지 않았다.
* 영상 표시 영역, 녹화, 감지, 파일 선택 재생, 드래그드롭 재생 로직은 변경하지 않았다.
* `README.txt`와 `docs/DECISIONS.md`에 Krypton Toolkit 도입 및 UI 테마 방향을 기록했다.
* `dotnet build DFBlackbox\DFBlackbox.csproj -o .buildcheck` 빌드가 경고 0개, 오류 0개로 성공했다.
